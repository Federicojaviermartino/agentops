# AgentOps - Deploy to Render

## Quick Deploy (5 minutes)

### Option A: Render Blueprint (recommended)

1. Push this repo to GitHub
2. Go to https://dashboard.render.com/select-repo?type=blueprint
3. Select your repo
4. Render reads `render.yaml` and creates:
   - Web service: `agentops-api` (Node.js, Frankfurt)
   - Database: `agentops-db` (PostgreSQL 16, Frankfurt)
5. Set environment variables in Render dashboard:
   - `ANTHROPIC_API_KEY` (required for AI agents)
   - `STRIPE_SECRET_KEY` (optional, for billing)
   - `RESEND_API_KEY` (optional, for emails)
6. Deploy triggers automatically

### Option B: Manual Setup

```bash
# 1. Create PostgreSQL database on Render
#    - Go to Dashboard > New > PostgreSQL
#    - Name: agentops-db, Region: Frankfurt, Plan: Starter
#    - Copy the Internal Database URL

# 2. Create Web Service
#    - Go to Dashboard > New > Web Service
#    - Connect your GitHub repo
#    - Settings:
#      Name: agentops-api
#      Region: Frankfurt
#      Runtime: Node
#      Build Command: npm ci && npx prisma generate --schema=src/prisma/schema.prisma && npm run build
#      Start Command: npx prisma migrate deploy --schema=src/prisma/schema.prisma && node dist/main.js
#      Plan: Starter ($7/mo)

# 3. Environment Variables (in Render dashboard):
DATABASE_URL=<from step 1>
NODE_ENV=production
JWT_SECRET=<auto-generated>
JWT_REFRESH_SECRET=<auto-generated>
FRONTEND_URL=https://agentops-api.onrender.com
ANTHROPIC_API_KEY=<your key>
ENABLE_DOCS=true
APP_VERSION=1.0.0
```

## After Deploy

### Seed demo data (first time only)

```bash
# SSH into Render shell or run from local with DATABASE_URL pointing to Render:
npx prisma db seed
# Or manually:
npx ts-node src/prisma/seed.ts
```

### Verify deployment

```bash
# Health check
curl https://agentops-api.onrender.com/health

# Readiness (DB connection)
curl https://agentops-api.onrender.com/health/ready

# Demo login (no auth required)
curl https://agentops-api.onrender.com/demo

# Demo token
curl -X POST https://agentops-api.onrender.com/demo/login

# API docs
open https://agentops-api.onrender.com/docs

# Landing page
open https://agentops-api.onrender.com/
```

### Test authenticated endpoints

```bash
# Login
TOKEN=$(curl -s -X POST https://agentops-api.onrender.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@techstart.es","password":"AgentOps2026!"}' \
  | jq -r '.accessToken')

# Get AI systems
curl -H "Authorization: Bearer $TOKEN" https://agentops-api.onrender.com/ai-systems

# Get compliance score
curl -H "Authorization: Bearer $TOKEN" https://agentops-api.onrender.com/compliance-score/organization

# Get benchmark
curl -H "Authorization: Bearer $TOKEN" https://agentops-api.onrender.com/benchmark

# Search
curl -H "Authorization: Bearer $TOKEN" "https://agentops-api.onrender.com/search?q=hiring"

# Ask assistant
curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"question":"What are the fines?"}' \
  https://agentops-api.onrender.com/assistant/ask
```

## Custom Domain

1. In Render dashboard > Settings > Custom Domains
2. Add: `api.agentops.eu` (or your domain)
3. Add CNAME record in your DNS:
   - `api.agentops.eu` -> `agentops-api.onrender.com`
4. Render auto-provisions SSL certificate
5. Update `FRONTEND_URL` env var to your new domain

## Monitoring

- Health: `/health` (basic)
- Ready: `/health/ready` (includes DB check)
- Logs: Render dashboard > Logs
- Metrics: Render dashboard > Metrics

## Local Development

```bash
# Clone
git clone <your-repo> && cd agentops-integrated

# Install
npm install

# Start infrastructure
docker compose -f docker/docker-compose.yml up -d

# Setup database
cp .env.example .env
npx prisma generate --schema=src/prisma/schema.prisma
npx prisma migrate dev --schema=src/prisma/schema.prisma --name init
npx ts-node src/prisma/seed.ts

# Run tests
npm test

# Start dev server
npm run start:dev

# Open
# Landing:  http://localhost:3000/
# API docs: http://localhost:3000/docs
# Demo:     http://localhost:3000/demo
# Login:    admin@techstart.es / AgentOps2026!
```
